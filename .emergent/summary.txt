**PRODUCT REQUIREMENTS (MVP Scope):**

    *   Automated lead creation and round-robin assignment.

    *   Create/edit venue listings with structured data, images, pricing, and amenities.
    *   Basic availability calendar.
    *   View and respond to enquiries.
5.  **Event Planner Dashboard**:
    *   Create and manage professional profiles with services, pricing, and portfolio.

**Code Architecture**


**Key Technical Concepts**
- **Backend**: FastAPI, MongoDB (with Motor for async operations), Pydantic models.
- **Frontend**: React, React Router, Tailwind CSS, Shadcn UI.
- **Authentication**: JWT for email/password auth, integration with Emergent-managed Google OAuth.
- **Architecture**: Monorepo with a decoupled frontend and backend. Role-based access control (RBAC) managed via private routes and auth context in React.

**key DB schema**
-   **users**: { email, password_hash, name, role: Enum['Customer', 'RelationshipManager', ...], phone, status }
-   **venues**: { name, city, area, coordinates: { lat, lon }, event_types, pricing: { per_plate, min_spend }, amenities, images, owner_id, status }
-   **leads**: { customer_details: { name, email, phone }, event_type, date, guests, budget, rm_id, stage: Enum['New', 'Contacted', ...], notes }
-   **event_planners**: { name, user_id, services, portfolio_images, status }
