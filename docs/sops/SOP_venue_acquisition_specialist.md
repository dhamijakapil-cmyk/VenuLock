# SOP: Venue Acquisition Specialist

**Role:** `venue_specialist`  
**Portal:** teams.venuloq.com → Field Workflow  
**Daily load:** High — field capture and data collection

---

## Responsibilities

1. Identify potential venue partners in assigned territory
2. Conduct Quick Captures (initial interest assessment)
3. Conduct Full Captures (detailed venue data collection)
4. Upload venue photos (minimum 3 for publish eligibility)
5. Record owner contact details, commercial interest, and meeting outcomes
6. Submit captures for Team Lead review
7. Address send-back feedback and resubmit

---

## Daily Workflow

### Quick Capture (`/team/field/quick`)
For first-touch venue discovery:
1. Enter venue name, city, locality
2. Record owner name and phone
3. Note initial interest level (hot/warm/cold)
4. Save as draft

### Full Capture (`/team/field/captures`)
For detailed venue assessment:
1. Fill ALL mandatory fields:
   - Venue name, owner name, owner phone
   - City, locality, venue type
   - Capacity min/max
2. Add commercial data:
   - Pricing band (min/max per plate)
   - Owner interest level
   - Meeting outcome
3. Upload 3+ photos
4. Add specialist notes (30+ characters recommended)
5. Record amenities and event types supported

### Submission
1. System validates mandatory fields before allowing submission
2. Click "Submit for Review" → status changes to `submitted_for_review`
3. Team Lead will review and either:
   - Route to Data Team for refinement
   - Send back with feedback

### Handling Send-Backs
When a capture is sent back (`sent_back_to_specialist`):
1. Read the send-back reason carefully
2. Fix the flagged issues
3. Resubmit for review

---

## Mandatory Fields Checklist

| Field | Required | Notes |
|-------|----------|-------|
| venue_name | ✅ | Min 5 chars, use proper casing |
| owner_name | ✅ | Full name of primary contact |
| owner_phone | ✅ | With country code if possible |
| city | ✅ | System auto-normalizes (e.g., gurgaon → Gurugram) |
| locality | ✅ | Specific area/neighborhood |
| venue_type | ✅ | Use standard types (banquet_hall, hotel, etc.) |
| capacity_min | ✅ | Minimum guest capacity |
| capacity_max | ✅ | Maximum guest capacity |

## Quality Tips
- Use proper capitalization (not ALL-CAPS, not all-lowercase)
- Include detailed specialist notes — these help write listing copy later
- Photos should show the venue clearly (no blurry/dark images)
- Record commercial model openness honestly
- Always note next follow-up date
